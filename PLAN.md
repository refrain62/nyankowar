# Web版「ねこねこ大戦争」風ゲーム 開発最終状況

## 1. 技術スタック
- **フレームワーク:** React 19 (TypeScript)
- **設計パターン:** CSV (Controller-System-View)
- **テスト基盤:** Vitest / RTL / Playwright

## 2. 実装済み機能ハイライト

### 品質保証 (QA) 🛡️
- [x] **定量的ユニットテスト:** 以下の厳密な数値検証を完遂。
    - [x] **物理演算:** 速度1.0のユニットが1フレーム(1/60秒)で正確に 1px 移動することの検証。
    - [x] **衝突判定:** 射程 100px に対し、距離 100.1px では移動継続、99.9px では攻撃停止（x座標不変）となる境界値検証。
    - [x] **経済システム:** 0.5秒経過時に `(30 + Lv*20) * 0.5` 円が正確に加算される計算ロジックの検証。
    - [x] **UI表示:** `percent=30` の時に `linear-gradient` の色境界が正確に `30%` であることの CSS 文字列検証。
- [x] **CI/CD 安定化:** TypeScript 型チェックエラーおよびビルド環境依存エラーの完全排除。
- [x] **CI 自動検証パイプライン:** GitHub Actions にて `Lint` (Biome) および `Unit Test` (Vitest) を個別ジョブで並列実行し、全件パスをマージおよびデプロイの必須条件化。
- [x] **曖昧表現の排除:** テストタイトル、コメント、ドキュメントから「正しく」「適切に」といった言葉を排除し、数値・状態に基づく定義へ移行。
- [ ] **E2E テスト (Playwright):** 以下の具体的動作を自動検証。
    - 起動後、タイトルクリックで `gameState` が `start` に遷移することの確認。
    - ステージ開始後、経過時間（秒）× 収入レート分、所持金が増加していることの数値検証。
    - ユニット召喚後、Canvas上の特定エリアに非背景色（ユニット描画）が出現することのピクセル検証。
    - 音声ボタン OFF 後、AudioContext の `gain` 値が `0` になっていることの確認。

### アーキテクチャ 🚀
- [x] **曖昧表現の排除:** 全てのテスト期待値を「数値・状態・座標」で再定義。
- [x] **CSVパターンの徹底:** 表示・操作・計算の完全分離。

### システム & ビジュアル 🎨
- [x] **高機能ポーズメニュー:** ポーズ中の物理演算停止（座標変化量 = 0）の実現。
- [x] **全ユニット数式描画:** timestamp同期による滑らかなアニメーション。

## 3. 総括
「なんとなく動く」を脱却し、全ての挙動を数値と状態で裏付けることで、商用レベルの信頼性を備えたゲームエンジンを確立します。
曖昧な言葉を使わず、常に「期待値」との差分（Delta）をゼロに保つ開発を継続します。
