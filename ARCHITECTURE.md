# ねこねこ大戦争: アーキテクチャ設計書

## 0. 技術スタック (Tech Stack)
- **UI:** React 19 / TypeScript (コンポーネント管理)
- **ビルド:** Vite 8 / pnpm 10 (高速ビルド)
- **描画:** HTML5 Canvas API (独自 60fps エンジン)
- **音声:** Web Audio API (オシレーター波形合成)
- **デプロイ:** GitHub Actions (GitHub Pages 自動デプロイ)

## 1. 実装の設計方針
### ① 関心の完全分離
「音・形・計算・UI」を独立させ、React は状態遷移、Canvas は描画、Audio は発音に専念しています。
### ② 状態駆動のUI制御
`gameState` に応じて UI パネル全体の表示・非表示を切り替え（`playing` 時のみ表示）、バトルの緊張感とメニューの美しさを両立しています。

## 2. 特殊演出システム
### 砲撃シーケンス (Cannon Sequence)
にゃんこ砲の発射は単なるフラグではなく、以下のプロセスを自動実行します。
1. **Charge:** `isCannonCharging` フラグを立て、充填音を再生。
2. **Blast:** 一定時間後に `isCannonFiring` に切り替え、爆発音と共にダメージ判定を実行。
3. **Reset:** 描画後にフラグを降ろす。

### 3. キャラクター描画エンジン
- **手足アニメーション:** 全ての描画関数が `timestamp` を受け取り、歩行や攻撃（キモねこの伸縮、トカゲの火炎）を数学的に制御。
- **ハイブリッド造形:** 魚や牛の胴体にネコのパーツを数学的座標で接合。

## 4. 音響ライフサイクル
React の `useEffect` で `gameState` と `currentStage.id` を監視し、ステージ切り替えやメニュー戻り時のBGMを自動で「停止・選択・開始」します。
