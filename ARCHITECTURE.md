# ねこねこ大戦争: アーキテクチャ設計書

## 0. 技術スタック (Tech Stack)
- **UI:** React 19 / TypeScript
- **描画:** HTML5 Canvas API (340px 高さに最適化)
- **音声:** Web Audio API (独自 8bit シンセ)
- **設計思想:** **Event-Driven FSM (useEffect-free Architecture)**

## 1. 実装の設計方針
### ① 画面レイアウトの最適化
画面高を 340px に固定し、ワイド画面での戦況把握を優先。地面（GroundY）の位置を基準として、全てのオブジェクトを相対的に配置しています。

### ② イベント駆動型状態管理 (FSM)
`transitionTo` 関数を介してゲーム状態を遷移させる **有限状態マシン (FSM)** を採用。
React の `useEffect` による「状態同期」を徹底的に排除し、状態の変化（イベント）に直接副作用（BGM開始・停止等）を紐付けることで、予測可能な挙動を保証します。

## 2. アニメーション・エンジニアリング
### ① 統一されたボーン構造
全てのネコユニットに共通の `timestamp` を渡し、歩行や特殊攻撃をフレーム単位ではなく時間単位で補間。これにより、ブラウザの負荷状況に関わらず一定の速度で滑らかな動きを実現。

### ② パーティクル・エフェクト
火炎放射、爆発などは画像を使わず、円形グラデーションと明滅（フリッカー）の数式的な組み合わせで表現。

## 3. 音響ライフサイクル (Stable Audio Interface)
### ① Stable Interface
`useGameAudio` カスタムフックは、`useMemo` を通じて完全に不変（Stable）なオブジェクトを返却します。これにより、ゲームループ（毎フレーム実行）に渡されるオーディオ制御オブジェクトの参照が固定され、パフォーマンスの最大化を図っています。

### ② 唯一の正当な useEffect
コンポーネントのアンマウント（不意の画面消失）時に BGM を停止するクリーンアップのみ、`useEffect` の責務として残しています。これは外部リソース（Web Audio API）の安全な破棄という、React における副作用の本来の目的です。

## 4. 特殊演出シーケンス
### 砲撃 (Cannon)
単なるダメージ処理ではなく、「充填（Sound & Lane Effect）」➔「発射（Sound & White-out）」➔「結果（Damage & Knockback）」という多段階の非同期処理を `stateRef` で管理。
