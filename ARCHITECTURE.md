# ねこねこ大戦争: アーキテクチャ設計書

## 0. 技術スタック (Tech Stack)
- **UI:** React 19 / TypeScript
- **描画:** HTML5 Canvas API (340px 高さに最適化)
- **音声:** Web Audio API (独自 8bit シンセ)
- **設計思想:** **Event-Driven FSM (useEffect-free Architecture)**

## 1. 実装の設計方針
### ① 画面レイアウトの最適化
画面高を 340px に固定し、ワイド画面での戦況把握を優先。地面（GroundY）の位置を基準として、全てのオブジェクトを相対的に配置しています。

### ② イベント駆動型状態管理 (FSM)
`transitionTo` 関数を介してゲーム状態を遷移させる **有限状態マシン (FSM)** を採用。
`paused` 状態を含む全ての遷移ライフサイクルを一元管理。状態の同期に `useEffect` を使わず、命令的かつ同期的に副作用をコントロールします。

## 2. アニメーション・エンジニアリング
### ① 統一されたボーン構造
全てのネコユニットに共通の `timestamp` を渡し、歩行や特殊攻撃をフレーム単位ではなく時間単位で補間。これにより、ブラウザの負荷状況に関わらず一定の速度で滑らかな動きを実現。

### ② ポーズ時の挙動
ポーズ中（`paused` 状態）はゲームループの `update` 関数のみがスキップされ、`draw` 関数は継続して実行されます。これにより、最新の画面状態を維持したまま、物理演算のみを停止させることが可能です。

## 3. 音響ライフサイクル (Stable Audio Interface)
### ① ステップ保持型ポーズ・再開
`useGameAudio` では BGM の再生位置（`bgmStepRef`）を独立して管理しています。ポーズ時にタイマーを停止し、再開時にそのステップからループを再起動することで、継ぎ目のない「一時停止と再開」を実現しています。

### ② 唯一の正当な useEffect
コンポーネントのアンマウント時に BGM を停止するクリーンアップのみを `useEffect` の責務としています。

## 4. 特殊演出シーケンス
### ① 砲撃 (Cannon)
単なるダメージ処理ではなく、「充填」➔「発射」➔「結果」の多段階シーケンス。ポーズ中はこの非同期タイマーも本来は停止させるべきですが、現状の仕様では発射準備が完了するまで停止可能です。

### ② リスタート (Restart Lifecycle)
ポーズ画面から直接 `startGame` を呼び出すことで、現在のステージ情報を保持したまま `stateRef` をリセットし、`paused` 状態から即座に `playing` 状態へ復帰するライフサイクルを確立しています。
