# ねこねこ大戦争: アーキテクチャ設計書

## 0. 技術スタック (Tech Stack)
- **UI:** React 19 / TypeScript
- **ビルド:** Vite 8 / pnpm 10
- **描画:** HTML5 Canvas API (340px 高さに最適化)
- **音声:** Web Audio API (独自 8bit シンセ)
- **デプロイ:** GitHub Actions

## 1. 実装の設計方針
### ① 画面レイアウトの最適化
画面高を 340px に固定し、ワイド画面での戦況把握を優先。これに伴い、地面（GroundY）の位置を動的に計算し、城やユニット、エフェクトの Y 座標を相対的に配置する設計を採用しています。

### ② 没入型 UI システム
`gameState` をキーとして、操作パネルの React DOM 要素を条件付きレンダリング。バトル開始時の `fadeIn` アニメーションにより、メニューから戦闘への切り替えをドラマチックに演出しています。

## 2. アニメーション・エンジニアリング
### ① 統一されたボーン構造
全てのネコユニット（`CatBasic`, `CatTank` 等）に共通の `timestamp` を渡し、歩行や特殊攻撃（火炎、ムチ腕）をフレーム単位ではなく時間単位で補間。これにより、ブラウザの負荷状況に関わらず一定の速度で滑らかな動きを実現。

### ② パーティクル・エフェクト
火炎放射（ネコトカゲ）は、複数の円形グラデーションを時間差で描画し、激しい明滅（フリッカー）を加えることで、画像アセットを使わずに熱量を表現。

## 3. 特殊演出シーケンス
### 砲撃 (Cannon)
単なるダメージ処理ではなく、「充填（Sound & Line Effect）」➔「発射（Sound & White-out）」➔「結果（Damage & Knockback）」という多段階の非同期処理（setTimeout）を stateRef で管理。

## 4. 音響ライフサイクル
React Effect が `gameState` と `currentStage.id` を監視し、ステージ選択 ➔ バトル ➔ メニュー戻り の各フェーズで適切な BGM を自動的にハンドリング。
