# ねこねこ大戦争: アーキテクチャ設計書

## 0. 技術スタック (Tech Stack)
本プロジェクトは、外部アセット（画像・音声ファイル）を一切使用せず、全てコードによる生成（Procedural Generation）で構築されています。

- **言語:** TypeScript (型安全な開発)
- **UIフレームワーク:** React 19 (コンポーネントベースのUI管理)
- **ビルドツール:** Vite 8 (高速な開発サーバーとビルド)
- **パッケージマネージャー:** pnpm 10 (高速かつ効率的な依存関係管理)
- **描画エンジン:** HTML5 Canvas API (独自のアニメーション・描画システム)
- **音声エンジン:** Web Audio API (オシレーターによる 8bit 風シンセサイザー)
- **デプロイ:** GitHub Actions (CI/CD による GitHub Pages への自動デプロイ)

## 1. 実装の設計方針 (Design Philosophy)

本ゲームは以下の3つの柱を重視して設計されています。

### ① 関心の完全分離 (Separation of Concerns)
「音を鳴らす」「形を描く」「物理演算する」「UIを表示する」という4つの責務を完全に独立させています。
- **Canvas:** 描画のみを担当。外部からの `timestamp` と `stateRef` を受け取り、それを愚直に画面に反映します。
- **Audio:** 再生ロジックのみを担当。React のライフサイクルから独立した `setTimeout` スケジューラを持ち、BGMの途絶を防ぎます。
- **React:** ステートとイベントハンドリングを担当。Canvas 内部の複雑な計算には関与せず、高レベルな画面遷移やボタンの有効化のみを制御します。

### ② データ駆動型アプローチ (Data-Driven Design)
キャラクターの性能やステージの構成は、コード内にハードコードせず、`constants/` 以下の定義ファイルに集約しています。
- 新しいユニットや敵、ステージを追加する際、主要なロジックファイルを書き換える必要はありません。定義ファイルを追加し、目録に登録するだけでシステムが自動的に拡張されます。

### ③ 演算と描画の同期最適化
一般的な React アプリケーションと異なり、ゲームとしての安定性を確保するため以下の手法を採用しています。
- **stateRef による非同期演算:** React の `useState` は描画のトリガーになるため、毎秒60回の更新には向きません。物理演算やタイマーは `useRef` で管理し、Canvas ループ内で直接更新することで、描画負荷を最小限に抑えています。
- **数学的アニメーション:** アニメーションを「フレーム数」ではなく「タイムスタンプ（時間経過）」に基づいた数学関数（Sine, Bezier）で定義しています。これにより、PCの性能に関わらず一定のスピードで滑らかな動きを実現しています。

## 2. フォルダ構造
- **`src/types/`**: システム全体のデータ構造の型定義。
- **`src/constants/`**: ユニット・ステージのマスターデータ。
- **`src/hooks/`**: 音声やライフサイクルの管理。
- **`src/audio/`**: 波形生成（core）、効果音（se）、演奏ロジック（music）。
- **`src/components/Canvas/`**: 各種 Renderer（units, stage）。
- **`src/components/UI/`**: GaugeButton 等の共通パーツ。

## 3. 音響システム (Audio Decoupling)
音声は「生成(Synth)」と「制御(Hook)」に分離されています。
`useGameAudio` は React Effect と連携し、`gameState` を監視して BGM を自動制御します。

## 4. 描画パイプライン (Visual Decoupling)
Canvas 描画はレイヤーごとに分割され、`renderer.ts` が司令塔として順序を制御します。
1. **Background** 2. **Castle** 3. **Units (Entity)** 4. **Effects** 5. **UI Overlay**

## 5. 安定化エンジン
`useRef` を中心とした状態管理により、React の再レンダリングサイクルの影響を受けずに、正確な 60fps 物理演算と BGM のタイミング制御（400ms単位）を実現しています。
